```
A0 ─ A1 ──────────────┐
                       ├─▶ E4(스테이션 편입) ─▶ M3 수직슬라이스
E0 ─ E1 ─ E2 ─ E3 ─────┘            │
                                    ├─ A2(IK) ─┘
                                    ▼
                         A3(티칭) ─ A4(웨이퍼) ─ E5(미션) ─▶ M4
                                    ▼
                         E6(장비 추가) ─ A6(안전/HMI) ─▶ M5
                                    ▼
                         A5(트윈 통신) ─ E7(폴리시/대시보드) ─▶ M6
```

**임계 경로(Critical Path):** `A1 → E3 → E4 → E5 → E6`
게임 루프를 막는 최단 경로. A5(통신)·A6(안전)·E7(폴리시)는 상대적으로 병렬/후순위.

## 상세 실행 순서

### M1 — 제어 코어  *(현재 위치)*
- **A0** 토대: DigitalTwin 씬·조명·카메라, `Build 6-Axis Robot` 메뉴 실행.
- **A1** FK+조그: 슬라이더 6축 검증, 축 방향 보정, TCP 기즈모(이미 반영됨).
- **게이트:** Play 시 조그 동작 + 콘솔 에러 0.

### M2 — 게임 셸
- **E0** `GameManager`/`GameState`/`EventBus`/Bootstrap + Additive 씬 로더.
- **E1** 로그인/프로필 *(로컬 프로필, 실인증 아님)* + 진행도 저장.
- **E2** 탐험: `PlayerController`(Input System) + 팔로우 카메라 + 클린룸 그레이박스.
- **E3** 상호작용: `Interactable` 근접 + "제어하시겠습니까?" 프롬프트 + 확인 이벤트.
- **게이트:** 로그인→걸어다님→장비 앞 프롬프트→확인 이벤트 발행.

### M3 — ★수직 슬라이스 (MVP, 가장 중요)
- **E4** Station Framework: `StationBase`/`StationDefinition`/`StationRegistry`,
  **6축 로봇을 `RobotStation` 으로 래핑**, 제어 씬 Additive 로드, 입력/카메라 전환.
- **A2** IK: DLS 수치해로 목표점 추종 (제어 모드 조작감 향상).
- **게이트:** 걸어가서 로봇 클릭→제어 진입→IK 조작→ESC 탐험 복귀. **핵심 루프 완성.**

### M4 — 교육 게임화
- **A3** 티칭/재생: 웨이포인트 + 사다리꼴 속도 프로파일.
- **A4** 웨이퍼 시나리오: FOUP↔챔버 픽/플레이스(reparent) + 상태머신 + 매핑.
- **E5** 미션/평가: `MissionDefinition`(SO) + `MissionRunner` + `Scorer` + 디브리핑 + 해금.
- **게이트:** "슬롯N 웨이퍼를 제한시간 내 무충돌 이송" 미션 수행→채점→피드백→진행 저장.

### M5 — 확장 실증
- **E6** 콘텐츠 확장: 장비 2~3종(얼라이너/식각/CMP 등)을 **SO+프리팹만으로** 편입.
- **A6** 안전/HMI: 충돌감지·세이프티존·E-stop·소프트리밋, 장비별 HMI 패널.
- **게이트:** 코어 코드 수정 없이 신규 장비/미션 추가 성공 + 안전 인터락 동작.

### M6 — 트윈 & 폴리시
- **A5** 디지털 트윈 통신: MQTT/TCP → OPC-UA → (반도체 특화) SECS/GEM,
  백그라운드 스레드 + `ConcurrentQueue` 메인스레드 마샬링.
- **E7** 폴리시: 미니맵·튜토리얼·사운드·일시정지 + **강사용 대시보드/분석 로깅** + 트윈 연동.
- **게이트:** 외부 컨트롤러와 실시간 미러링 + 학생별 학습 데이터 수집.

## 운영 규칙 (MCP 세션)
- 각 마일스톤 = **작동하는 데모**로 끝난다(가로 슬라이스 우선, 완벽주의 지양).
- Phase 완료 기준(게이트) 충족 → **씬/스크립트 커밋** → 다음.
- 스크립트 변경 후 `read_console` 에러 0 확인. 씬 변경 후 저장.
- 상세 Phase 내용은 각 트랙 문서(DIGITAL_TWIN_PLAN / EXPERIENCE_LAYER_PLAN) 참조.
