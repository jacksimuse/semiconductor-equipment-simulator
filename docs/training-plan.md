# 교육 운영안

이 문서는 반도체 장비 디지털 트윈 교육 시뮬레이터를 활용한 수업 흐름 예시입니다.

## 교육 대상

- 마이스터고, 특성화고, 전문대, 대학의 반도체 및 메카트로닉스 관련 수업
- 기업 신입 엔지니어와 직무 전환 교육 대상자
- 장비 자동화, 로봇, 설비 제어 입문자

## 학습 목표

- 반도체 장비 내부의 기본 구성과 자동화 흐름을 설명할 수 있다.
- 6축 로봇 조그와 티칭 개념을 이해하고 직접 조작할 수 있다.
- 웨이퍼 이송 시나리오를 실행하고 주요 상태 변화를 관찰할 수 있다.
- 충돌 감지와 E-STOP 같은 안전 인터락의 필요성을 설명할 수 있다.
- 제공된 Unity 프로젝트를 수정해 간단한 장비 시나리오를 확장할 수 있다.

## 1일 과정 예시

| 시간 | 주제 | 실습 |
| --- | --- | --- |
| 1교시 | 반도체 장비 자동화 개요 | 장비 구성 요소와 로봇 이송 흐름 살펴보기 |
| 2교시 | Unity 프로젝트 실행 | 프로젝트 열기, 씬 실행, 조작 UI 확인 |
| 3교시 | 로봇 조그와 티칭 | 관절 조그, 포즈 저장, 재생 흐름 실습 |
| 4교시 | 웨이퍼 이송 시나리오 | FOUP, 로봇, 챔버 간 이동 흐름 관찰 |
| 5교시 | 안전 인터락 | 충돌 감지와 E-STOP 동작 확인 |
| 6교시 | 미니 프로젝트 | 장비 배치 또는 시나리오 일부 변경 |

## 평가 과제 예시

- 로봇 조그 UI로 지정 포즈를 만들고 티칭 포인트로 저장하기
- 웨이퍼 이송 시나리오의 각 단계를 순서도로 설명하기
- E-STOP이 필요한 상황을 3가지 이상 작성하기
- 씬 안의 장비 배치 또는 장애물 위치를 바꾸고 충돌 여부를 관찰하기
- 신규 실습 시나리오 아이디어를 기획서 형태로 제출하기

## 강사용 운영 메모

- 첫 수업에서는 Unity 코드 수정이 아니라 실행과 관찰에 집중하는 편이 좋습니다.
- 학생 실습 PC에서는 Unity 버전과 프로젝트 경로에 한글/공백이 있어도 열리는지 사전 확인합니다.
- 코드 실습까지 진행할 경우 `Assets/DigitalTwin/Scripts/Runtime` 폴더의 런타임 스크립트를 중심으로 설명합니다.
- 학교 납품 시에는 학생용 매뉴얼과 강사용 해설 자료를 분리해 제공하는 구성이 좋습니다.
